# SGX OneBox Platform 平台使用说明

## 1. 文档范围

本文档基于当前代码实现整理，覆盖以下实际可用页面与操作：

- 仪表板
- 集群管理
- 安全组件管理
- 安全策略
- 飞地环境管理
- 合规管理
- 系统管理

本文档同时标注当前实现边界，便于现场交付、演示和培训统一口径。

## 2. 默认账号与角色

默认账号来自 `internal/store/seed.go`：

| 角色 | 用户名 | 初始密码 | 主要可见菜单 |
|---|---|---|---|
| 平台管理员 | `admin` | `admin123` | 仪表板、集群管理、安全组件管理、飞地环境管理、安全策略、合规管理、系统管理 |
| 安全管理员 | `security-admin` | `secure123` | 仪表板、集群管理、安全组件管理、飞地环境管理、安全策略、合规管理 |
| 审计员 | `auditor` | `audit123` | 仪表板、飞地环境管理、合规管理 |
| 运维员 | `operator` | `ops123` | 仪表板、集群管理、安全组件管理、安全策略 |

建议首次登录后先进入“系统管理 > 账号管理”修改当前账号密码。

## 3. 平台整体使用顺序

建议按下面顺序使用平台：

1. 登录平台，确认当前角色和可见菜单。
2. 在“集群管理”确认节点在线、资源余量、告警和日志状态。
3. 在“安全组件管理 > 镜像资产”登记镜像元数据并确认签名、SBOM、漏洞等级、飞地适配状态。
4. 在“安全组件管理 > 安装包”登记安装资源包，在“应用市场”维护可交付组件版本。
5. 在“安全策略 > 网络分区”创建策略域，在“策略规则”建立规则。
6. 在“安全组件管理 > 组件运维”创建组件，绑定镜像、命名空间、隔离级别和网络。
7. 组件需要飞地运行时，进入“飞地环境管理 > 飞地配置”补齐 MRENCLAVE、MRSIGNER、Quote Policy、Secrets Policy 和 SGX 设备。
8. 回到“组件运维”执行部署、生成 Manifest、升级、扩缩容。
9. 进入“飞地环境管理 > 远程证明”执行证明，必要时执行“安全巡检”。
10. 进入“合规管理”执行合规检查、导出报告、创建整改任务。
11. 进入“系统管理 > 审计日志”和“平台提示”复核证据与操作记录。

## 4. 登录与首页使用

### 4.1 登录

登录页输入用户名和密码后点击“进入平台”。登录成功后平台会保存 Token，并自动加载当前用户和仪表板快照。

### 4.2 仪表板

仪表板用于总览全局状态，重点查看：

- 镜像资产数量
- 运行负载数量
- 飞地配置数量
- 合规得分
- 在线节点数量
- 未关闭告警数量
- 集群资源图表
- 风险分布图表
- 合规趋势图表

适用场景：

- 交付上线前做整体巡检
- 日常值班时快速判断平台是否稳定
- 演示时展示平台全局能力

## 5. 集群管理

### 5.1 节点与告警

入口：`集群管理 > 节点与告警`

可执行操作：

1. 查看节点名称、角色、状态、版本、CPU、内存、Pod 数和心跳时间。
2. 按级别筛选告警。
3. 配置 CPU、内存、Pod 数阈值。
4. 点击“保存阈值”写入系统参数。

落地建议：

- 交付验收时先确认所有节点状态为 `ready`。
- 出现 `warning` 或 `critical` 告警时，先记录源节点和时间，再进入“运行日志”交叉排查。

### 5.2 资源配额

入口：`集群管理 > 资源配额`

可执行操作：

1. 新建或更新配额：填写配额 ID、作用域、CPU 限额、内存限额、Pod 限额。
2. 点击“保存配额”。
3. 查看每个作用域的 CPU/内存使用进度条和 Pod 配额。

建议做法：

- 每个命名空间或业务域单独建立一个作用域。
- 飞地相关组件建议单独使用一个受控作用域，便于资源隔离和审计。

### 5.3 运行日志

入口：`集群管理 > 运行日志`

可执行操作：

1. 输入关键字搜索日志内容。
2. 按 `error`、`warning`、`info` 过滤。
3. 根据 `nodeId / 时间 / category` 定位异常事件。

建议做法：

- 部署失败先筛 `warning` 和 `error`。
- 升级后先搜索 `scheduler`、`runtime` 类日志确认调度结果。

### 5.4 版本升级

入口：`集群管理 > 版本升级`

可执行操作：

1. 输入目标版本。
2. 输入升级通道。
3. 点击“执行升级”。

建议做法：

- 升级前先记录当前版本、导出合规报告、确认节点与配额正常。
- 升级后回到“节点与告警”与“运行日志”做复核。

## 6. 安全组件管理

### 6.1 镜像资产

入口：`安全组件管理 > 镜像资产`

页面分两部分：组件标准化建模、镜像构建与合规校验。

#### 6.1.1 组件标准化建模

这里展示平台内置的组件目录模板，包括：

- 组件名称
- 分类
- 厂商
- 版本
- 推荐隔离级别
- 依赖关系
- 描述

使用方式：

1. 先阅读模板项，确定组件适合使用 `standard`、`hardened` 还是 `enclave`。
2. 记录该组件依赖的网络或服务，后续创建组件和网络时使用。

#### 6.1.2 镜像构建与合规校验

字段含义：

- `镜像 ID`：平台内部唯一标识，后续组件创建时通过它引用镜像。
- `镜像名称`：页面显示名称。
- `Registry`：镜像仓库地址，例如 `registry.local`。
- `Repository`：仓库内镜像路径，例如 `security/waf`。
- `Tag`：版本标签，例如 `2.3.1`。
- `Digest`：镜像摘要，用于精确锁定具体内容版本。
- `漏洞等级`：镜像安全风险级别。
- `签名`：镜像是否已签名。
- `SBOM`：镜像是否有软件物料清单。
- `飞地适配`：镜像是否适配 SGX / 飞地场景。

当前必填项：

- `* 镜像 ID`
- `* 镜像名称`
- `* Registry`
- `* Repository`
- `* Tag`

当前实现边界：

- 该页面当前执行的是镜像元数据登记。
- 页面会把镜像信息写入平台数据库。
- 页面当前没有真实执行镜像下载、同步或拉取。

落地操作：

1. 填写镜像基础信息。
2. 按实际情况设置漏洞等级、签名、SBOM、飞地适配。
3. 点击“导入镜像”。
4. 在列表中确认 `Registry/Repository:Tag`、扫描时间、签名状态。

建议做法：

- 用统一命名规范维护镜像 ID，例如 `img-业务名`。
- 需要飞地运行的镜像勾选“飞地适配”。
- 签名和 SBOM 建议保持开启，便于合规检查拿到更高分。

### 6.2 组件运维

入口：`安全组件管理 > 组件运维`

用途：创建组件、修改版本、扩缩容、部署、删除、生成 Manifest。

创建组件操作：

1. 填写组件 ID 和组件名称。
2. 在“选择镜像”中选择已经登记的镜像。
3. 填写版本、命名空间、副本数。
4. 选择隔离级别：`standard`、`hardened`、`enclave`。
5. 填写网络 ID，多个网络用逗号分隔。
6. 点击“保存组件”。

部署与运维操作：

1. 点击“部署”将组件状态切换为运行态。
2. 点击“Manifest”查看当前组件的 Deployment YAML。
3. 输入目标版本后点击“升级”。
4. 输入副本数后点击“扩缩”。
5. 点击“删除”移除组件。

关键规则：

- 组件创建时必须引用已存在镜像。
- 组件填写的网络 ID 必须已在“网络分区”中存在。
- 副本数必须大于 0。
- `enclave` 组件部署前必须先存在飞地配置。

Manifest 使用方式：

1. 点击“Manifest”。
2. 页面底部会显示 Deployment YAML。
3. 用它做人工复核、交付说明或外部集群对接参考。

当前实现边界：

- Manifest 会根据网络挂载生成 `k8s.v1.cni.cncf.io/networks` 注解。
- `enclave` 组件会带 `runtimeClassName: enclave` 和 SGX 节点选择条件。
- 页面当前未展示 Service、Ingress、NodePort 等直接访问入口。

### 6.3 安装包

入口：`安全组件管理 > 安装包`

用途：登记交付安装介质或离线资源包。

可执行操作：

1. 填写安装包 ID、安装包名称、模式、版本。
2. 设置是否签名、是否为离线包。
3. 点击“导入资源包”。
4. 在列表里确认模式、版本、签名状态。

模式说明：

- `iso`：整机或基础环境安装介质。
- `incremental`：增量更新包。
- `offline_bundle`：离线资源包。

建议做法：

- 交付现场优先维护离线包登记，便于审计和追溯。
- 组件升级时同步确认关联资源包版本。

### 6.4 应用市场

入口：`安全组件管理 > 应用市场`

用途：维护可发布的安全组件条目、历史版本、上架状态。

上架准备操作：

1. 填写组件 ID、组件名称、分类、当前版本、厂商。
2. 选择状态：`draft`、`published`、`unpublished`。
3. 填写依赖、适配环境、上传包名称、历史版本、说明。
4. 点击“上传到应用市场”。

版本与状态维护：

1. 在已有条目上输入新版本号和新上传包名称。
2. 点击“新增版本”。
3. 点击“上架”或“下架”。
4. 点击“删除”移除条目。

建议做法：

- 先把条目保存为 `draft`，确认依赖、环境和上传包后再上架。
- 每次发版都补充版本历史，便于审计和回滚说明。

## 7. 安全策略

### 7.1 策略规则

入口：`安全策略 > 策略规则`

可执行操作：

1. 填写规则 ID、规则名称、分类、作用域。
2. 选择模式：`enforce` 或 `monitor`。
3. 选择状态：`enabled` 或 `staged`。
4. 填写目标列表，多个目标用逗号分隔。
5. 点击“保存规则”。

使用建议：

- `monitor` 适合策略验证阶段。
- `enforce` 适合正式收敛后的强制策略。
- 合规检查会关注 `enforce + staged` 这类待生效规则。

### 7.2 网络分区

入口：`安全策略 > 网络分区`

可执行操作：

1. 填写网络 ID、名称、Bridge 名称、父网卡、VLAN ID、子网、网关。
2. 点击“保存策略域”。
3. 后续在“组件运维”把组件挂载到该网络。

使用建议：

- 外联流量、管理流量、核心区流量拆成不同策略域。
- VLAN 规划和命名规则先统一，再录入平台。

### 7.3 拓扑视图

入口：`安全策略 > 拓扑视图`

当前展示逻辑：

- 平台根据“网络分区”和“策略规则”自动生成可视化拓扑。
- 图中体现“网络 -> 策略”的关联关系。

当前实现边界：

- 页面当前是只读可视化。
- 页面当前没有拖拽设备、拖拽节点、手动连线编辑能力。

落地建议：

- 把拓扑视图当作策略关系复核界面使用。
- 真实拓扑设计先在外部网络方案中定稿，再录入平台中的网络与规则。

## 8. 飞地环境管理

### 8.1 飞地资源

入口：`飞地环境管理 > 飞地资源`

可执行操作：

1. 查看每个节点的 EPC 总量、已使用量、飞地数量、状态。
2. 判断飞地节点是否具备继续承载能力。

建议做法：

- 部署飞地组件前先看 EPC 余量。
- 节点状态为 `healthy` 时优先承载关键飞地组件。

### 8.2 飞地配置

入口：`飞地环境管理 > 飞地配置`

用途：为 `enclave` 组件绑定可信执行参数。

操作步骤：

1. 填写飞地配置 ID。
2. 选择一个隔离级别为 `enclave` 的组件。
3. 填写 `MRENCLAVE`、`MRSIGNER`。
4. 填写 `Quote Policy` 和 `Secrets Policy`。
5. 填写 SGX 设备列表，多个值用逗号分隔。
6. 点击“保存策略”。

关键规则：

- 飞地配置只能绑定 `enclave` 组件。
- 没有飞地配置的 `enclave` 组件无法部署。

### 8.3 远程证明

入口：`飞地环境管理 > 远程证明`

可执行操作：

1. 点击“执行远程证明”。
2. 查看每个组件的证明状态、控制项、策略结果、证据和时间。

当前实现边界：

- 页面标识为“演示模式”。
- 证明结果由平台内建逻辑生成。
- 组件存在飞地配置时会得到 `verified` 结果并释放密钥标志。

使用建议：

- 每次飞地组件部署后都执行一次远程证明。
- 合规报告前再次执行证明，保证证据时间接近检查时间。

### 8.4 安全巡检

入口：`飞地环境管理 > 安全巡检`

可执行操作：

1. 点击“执行可信巡检”。
2. 查看每个目标的状态、摘要和检查时间。

适用场景：

- 例行巡检
- 合规自查
- 飞地资源波动后的复核

### 8.5 加密密钥生命周期

入口：`飞地环境管理 > 密钥管理`

可执行操作：

1. 填写密钥 ID、密钥名称。
2. 选择组件。
3. 填写算法与状态。
4. 点击“保存密钥”。
5. 需要时可删除密钥条目。

建议做法：

- 关键密钥绑定到飞地组件。
- 密钥状态建议配合远程证明和巡检结果同步维护。

## 9. 合规管理

### 9.1 合规检查

入口：`合规管理 > 合规检查`

可执行操作：

1. 点击“执行合规检查”。
2. 查看合规报告标题、标准、得分、生成时间。
3. 查看每个控制项的风险等级、问题描述、整改建议。
4. 点击“导出报告 (JSON)”下载当前报告。

评分关注项来自当前实现：

- 镜像是否签名
- 镜像是否带 SBOM
- 镜像是否有高危漏洞
- 飞地组件是否有飞地配置
- 飞地组件是否完成有效证明
- 网络是否已挂载组件
- 策略是否待发布
- 组件是否采用推荐隔离级别

建议做法：

- 每次镜像导入、组件部署、策略更新后执行一次合规检查。
- 对高风险项优先创建整改任务。

### 9.2 策略与整改焦点

入口：`合规管理 > 策略与整改`

用途：快速查看重点策略和重点整改任务。

适用场景：

- 例会汇报
- 周期性风险复盘
- 审计前快速浏览重点事项

### 9.3 整改任务

入口：`合规管理 > 整改任务`

可执行操作：

1. 填写任务 ID、控制项 ID、控制项名称、责任人。
2. 选择任务状态：`tracking`、`blocked`、`done`。
3. 选择截止时间。
4. 填写任务说明。
5. 点击“保存任务”。
6. 需要时删除任务。

建议做法：

- 每个高风险控制项创建独立任务。
- 责任人统一使用平台账号名，便于追溯。

### 9.4 报告历史

入口：`合规管理 > 报告历史`

用途：查看历史检查记录、时间和得分变化。

使用建议：

- 每次大版本变更前后各跑一次检查。
- 用历史记录展示治理改善趋势。

## 10. 系统管理

### 10.1 账号管理

入口：`系统管理 > 账号管理`

功能分两块：当前账号改密、平台账号维护。

当前账号改密：

1. 输入当前密码。
2. 输入新密码。
3. 点击“修改密码”。

平台账号维护：

1. 填写用户 ID、用户名、显示名称。
2. 选择角色：`platform_admin`、`security_admin`、`auditor`、`operator`。
3. 选择状态：`active`、`disabled`。
4. 设置初始密码。
5. 点击“保存账号”。
6. 在列表中查看最近登录时间，必要时删除账号。

建议做法：

- 严格按最小权限分配角色。
- 现场演示结束后及时停用临时账号。

### 10.2 系统参数

入口：`系统管理 > 系统设置`

可执行操作：

1. 填写设置 ID、分类、参数名称、参数值。
2. 点击“保存参数”。
3. 在下方列表核对更新时间。

当前代码里已有的典型参数：

- 会话超时时间
- 审计保留周期
- 告警阈值相关参数

### 10.3 审计日志

入口：`系统管理 > 审计日志`

可执行操作：

1. 查看按操作类型聚合的统计图。
2. 按关键字搜索操作人、动作、目标。
3. 按动作类型筛选审计记录。
4. 清空筛选后恢复全量视图。

建议做法：

- 每次演示完成后筛选 `deploy-component`、`upgrade-component`、`attestation-run` 等动作做复核。
- 交付时把关键审计记录与合规报告一起归档。

### 10.4 平台提示

入口：`系统管理 > 平台提示`

用途：查看平台内建提示和最近审计动作。

当前提示内容包括：

- 远程证明与合规报表的证据说明
- `enclave` 组件的补充策略提示
- Bridge 网络与 VLAN 的配置提示
- 飞地组件处于 `draft` 的常见原因说明

适用场景：

- 新人培训
- 现场答疑
- 交付说明补充

## 11. 常见落地流程

### 11.1 新增一个标准组件

1. 进入“安全组件管理 > 镜像资产”，登记镜像。
2. 进入“安全策略 > 网络分区”，创建需要的网络。
3. 进入“安全策略 > 策略规则”，创建运行时或网络策略。
4. 进入“安全组件管理 > 组件运维”，创建组件并选择 `standard` 或 `hardened`。
5. 点击“部署”。
6. 点击“Manifest”复核编排结果。
7. 进入“合规管理 > 合规检查”执行一次检查。

### 11.2 新增一个飞地组件

1. 先登记镜像，并勾选“飞地适配”。
2. 创建组件时把隔离级别设为 `enclave`。
3. 在“网络分区”准备好所需策略域。
4. 进入“飞地环境管理 > 飞地配置”补齐 MRENCLAVE、MRSIGNER、Quote Policy、Secrets Policy、SGX 设备。
5. 回到“组件运维”点击“部署”。
6. 进入“飞地环境管理 > 远程证明”执行证明。
7. 进入“安全巡检”执行巡检。
8. 进入“合规管理 > 合规检查”输出报告。

### 11.3 发布一个应用市场条目

1. 在“安装包”登记交付包。
2. 在“应用市场”录入组件名称、分类、版本、厂商、依赖、适配环境、上传包名称。
3. 初始状态保存为 `draft`。
4. 在条目中补充历史版本和新包名称。
5. 点击“上架”。
6. 如需撤回，点击“下架”。

### 11.4 处理一次风险整改

1. 先执行“合规检查”。
2. 记录高风险控制项和建议。
3. 进入“整改任务”创建责任人和截止时间。
4. 如风险和策略相关，去“安全策略”调整规则。
5. 如风险和镜像相关，去“镜像资产”补签名、补 SBOM 或替换版本。
6. 再次执行合规检查，确认得分变化。
7. 在“审计日志”确认整改动作已留痕。

## 12. 当前实现边界

以下说明有助于正确理解平台现状：

1. 镜像“导入”当前执行的是元数据登记，页面当前没有真实镜像下载或同步能力。
2. 拓扑视图当前是只读关系图，页面当前没有拖拽编辑和手工连线能力。
3. 组件访问入口当前主要通过 Manifest 和网络挂载关系体现，页面当前没有 Service / Ingress 管理界面。
4. 远程证明和巡检当前以平台演示逻辑为主，适合演示、培训、联调和流程验证。
5. 合规报告支持 JSON 导出，当前页面没有 PDF 或 Word 导出入口。

## 13. 现场演示推荐脚本

## 14. 操作复核清单

每天值班建议按以下顺序复核：

1. 仪表板查看得分、告警、节点状态。
2. 集群管理查看告警和日志。
3. 飞地资源查看 EPC 余量。
4. 远程证明查看关键组件状态。
5. 合规管理查看是否存在高风险项。
6. 审计日志查看当日关键动作。

每次发版建议按以下顺序复核：

1. 镜像资产确认版本、签名、SBOM、漏洞等级。
2. 安装包和应用市场确认版本记录。
3. 组件运维执行升级或部署。
4. Manifest 人工复核。
5. 远程证明与巡检。
6. 合规检查和报告导出。
7. 审计日志归档。
